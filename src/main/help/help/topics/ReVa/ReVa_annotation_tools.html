ReVa - Comment & Bookmark Tools

The annotation tools allow you to add, retrieve, search, and remove comments and bookmarks in a Ghidra program. Comments provide textual annotations at specific addresses, while bookmarks mark locations of interest with categorized, searchable markers. Both are essential for documenting analysis findings and tracking progress during reverse engineering.

Comment Tools

Ghidra supports five types of comments, each displayed in a different location relative to the code unit in the listing view:

TypeValueDisplay Location
Pre-comment"pre"Above the code unit, on its own line.
End-of-line comment"eol"At the end of the code unit's line.
Post-comment"post"Below the code unit, on its own line.
Plate comment"plate"In a bordered box above the code unit (typically at function entries).
Repeatable comment"repeatable"Displayed at the code unit and at all addresses that reference it.

set-comment

Sets or updates a comment at a specific address. If a comment of the same type already exists at the address, it is overwritten with the new text.

Parameters

NameTypeRequiredDefaultDescription
programPath string Yes — Path to the program in the Ghidra Project.
addressOrSymbol string Yes — Address or symbol name where the comment should be placed.
comment string Yes — The comment text to set.
commentType string No "eol" Type of comment: "pre", "eol", "post", "plate", or "repeatable".

Return Value

FieldTypeDescription
successbooleantrue on success.
addressstringFormatted address where the comment was set.
commentTypestringType of comment that was set.
commentstringThe comment text.

An address can have one comment of each type simultaneously (e.g., both a "pre" and an "eol" comment). Setting a comment of one type does not affect comments of other types at the same address. The operation runs within a Ghidra transaction and rolls back on failure.

get-comments

Retrieves comments at a single address or within an address range. Can filter by comment type to retrieve only specific kinds of comments.

Parameters

NameTypeRequiredDefaultDescription
programPath string Yes — Path to the program in the Ghidra Project.
addressOrSymbol string No* — Address or symbol name to get comments from.
addressRange object No* — Address range with "start" and "end" properties.
commentTypes array No all types Array of comment type strings to retrieve (e.g., ["pre", "eol"]). If omitted, all five types are checked.

* Either addressOrSymbol or addressRange must be provided. An error is returned if neither is specified.

Return Value

FieldTypeDescription
commentsarrayArray of comment objects.
countintegerNumber of comments found.

Each comment object contains:

FieldTypeDescription
addressstringFormatted address.
commentTypestringType of comment.
commentstringComment text.

Only non-empty comments are returned. When using an address range, the tool iterates through all code units in the range and checks each for comments of the requested types.

remove-comment

Removes a specific comment at an address. Only the specified comment type is removed; other comment types at the same address are unaffected.

Parameters

NameTypeRequiredDescription

commentType string Yes Type of comment to remove: "pre", "eol", "post", "plate", or "repeatable".

Return Value

FieldTypeDescription

Internally, removal is performed by setting the comment to null. The operation runs within a Ghidra transaction and rolls back on failure.

search-comments

Searches for comments containing specific text across the entire program. Supports case-sensitive and case-insensitive matching, and can be filtered by comment type.

commentTypes array No all types Array of comment type strings to search within.

Return Value

FieldTypeDescription

countintegerNumber of results found.
maxResultsintegerMaximum result limit applied.

Each result object contains:

FieldTypeDescription

commentTypestringType of the matching comment.

codeUnitstringThe instruction or data at the comment's address (e.g., "PUSH EBP").

The search uses Ghidra's AddressIterator for memory-efficient traversal. It stops early once maxResults is reached. If no commentTypes filter is specified, all five comment types are searched. The codeUnit field provides context for where the comment appears.

Bookmark Tools

Bookmarks are categorized markers placed at addresses of interest. Unlike comments, which are textual annotations tied to comment types, bookmarks have a user-defined type (e.g., "Note", "Warning", "TODO") and an optional category for further organization. Bookmarks appear in Ghidra's Bookmark window and can be filtered, sorted, and navigated.

set-bookmark

Creates or updates a bookmark at a specific address. If a bookmark with the same type and category already exists at the address, it is replaced.

Parameters

NameTypeRequiredDefaultDescription
programPath string Yes — Path to the program in the Ghidra Project.
addressOrSymbol string Yes — Address or symbol name where the bookmark should be placed.
type string Yes — Bookmark type. Common types include "Note", "Warning", "TODO", "Bug", and "Analysis", but any string is valid.
comment string Yes — Bookmark comment text.
category string No "" Bookmark category for organizational grouping (e.g., "Crypto", "Network").

Return Value

FieldTypeDescription
successbooleantrue on success.
idintegerUnique bookmark ID assigned by Ghidra.
addressstringFormatted address.
typestringBookmark type.
categorystringBookmark category.
commentstringBookmark comment text.

A bookmark's type/category pair is unique per address. If a bookmark with the same type and category already exists at the address, it is automatically removed before the new one is created. An address can have multiple bookmarks of different types or categories.

remove-bookmark

Removes a specific bookmark identified by its address, type, and category.

Parameters

NameTypeRequiredDefaultDescription
programPath string Yes — Path to the program in the Ghidra Project.
addressOrSymbol string Yes — Address or symbol name of the bookmark to remove.
type string Yes — Bookmark type (must match exactly).
category string No "" Bookmark category (must match exactly).

Return Value

FieldTypeDescription

The bookmark is identified by the combination of address, type, and category. All three must match for the bookmark to be found and removed. An error is returned if no matching bookmark exists. The operation runs within a Ghidra transaction and rolls back on failure.

search-bookmarks

Searches for bookmarks matching various criteria. Supports filtering by text content, bookmark type, category, and address range. All filters are combined with AND logic.

Parameters

NameTypeRequiredDefaultDescription
programPath string Yes — Path to the program in the Ghidra Project.
searchText string No — Text to search for in bookmark comments (case-insensitive substring match).
types array No — Filter by bookmark types (e.g., ["Note", "TODO"]). Empty or omitted means no type filtering.
categories array No — Filter by bookmark categories (e.g., ["Crypto", "Network"]). Empty or omitted means no category filtering.
addressRange object No — Limit search to an address range with "start" and "end" properties.
maxResults integer No 100 Maximum number of results to return.

Return Value

FieldTypeDescription
resultsarrayArray of matching bookmark objects (same structure as get-bookmarks).
countintegerNumber of results found.
maxResultsintegerMaximum result limit applied.

Filter Logic

All provided filters are combined with AND logic. A bookmark must satisfy every active filter to be included in results:

  • Address range: Bookmark must be within the specified range.
  • Types: Bookmark type must be in the provided list.
  • Categories: Bookmark category must be in the provided list.
  • Search text: Bookmark comment must contain the text (case-insensitive).

Omitting a filter (or providing an empty list) disables that filter — it does not mean "match nothing".

list-bookmark-categories

Lists all categories used by bookmarks of a given type, along with the count of bookmarks in each category. Useful for discovering how bookmarks are organized before searching.

Parameters

NameTypeRequiredDescription

type string Yes Bookmark type to list categories for (e.g., "Note").

Return Value

FieldTypeDescription
typestringThe bookmark type queried.
categoriesarrayArray of category objects.

Each category object contains:

FieldTypeDescription

Bookmarks with null or empty categories are normalized to "(no category)" in the output. If the specified type has no bookmarks, an empty category list is returned (not an error).

Tips & Best Practices

  • Use plate comments for function-level documentation and EOL comments for inline annotations on individual instructions.
  • Use repeatable comments for data items that are referenced from many places — the comment will automatically appear at each reference location in the listing.
  • Use bookmarks with consistent types and categories to create a searchable taxonomy of analysis findings (e.g., type "TODO" with categories like "Rename", "Investigate", "Suspicious").
  • Use list-bookmark-categories to understand how bookmarks are organized before searching with search-bookmarks.
  • Comments and bookmarks complement each other: comments provide detailed textual annotations visible in the listing, while bookmarks provide navigable markers visible in Ghidra's Bookmark window.
  • Use search-comments to find previous analysis notes, especially when picking up work from a prior session.
  • Remember that each address can have one comment of each type simultaneously, but bookmark uniqueness is determined by the type + category pair.
