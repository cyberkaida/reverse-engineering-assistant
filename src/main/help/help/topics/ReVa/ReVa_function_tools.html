ReVa Function Management Tools

The ReVa function management tools provide MCP-accessible interfaces for listing, searching, creating, and modifying functions in a Ghidra program. These tools support pagination for large binaries, similarity-based search, prototype editing, tag management, and discovery of undefined function candidates.

get-function-count

Return the total number of functions in a program. Use this tool before paginating through functions with get-functions to determine the total result set size.

Parameters

Parameter Type Required Default Description
programPath string Yes — Path in the Ghidra Project to the program.
filterDefaultNames boolean No true Exclude functions with Ghidra-generated default names (FUN_, DAT_, etc.) from the count.

Return Value

Returns a JSON object containing:

  • count — Total number of functions matching the filter criteria.
  • success — Boolean indicating the operation completed.

A warning is logged if this operation takes longer than 5 seconds, which may occur on very large binaries.

get-functions

List functions in a program with pagination, filtering, and optional verbose output. In compact mode (default), returns a summary of each function. In verbose mode, returns full details including signatures, parameters, and calling convention.

Parameters

Parameter Type Required Default Description
programPath string Yes — Path in the Ghidra Project to the program.
filterDefaultNames boolean No true Exclude functions with Ghidra-generated default names.
filterByTag string No — Only return functions that have this specific tag. Mutually exclusive with untagged.
untagged boolean No false Only return functions that have no tags. Mutually exclusive with filterByTag.
verbose boolean No false Return full function details instead of compact summaries.
startIndex integer No 0 Pagination start index (0-based).
maxCount integer No 100 Maximum number of functions to return per page. Recommended: 100 or fewer.

Return Value

Returns a multi-JSON response. The first object is a metadata header:

  • startIndex, requestedCount, actualCount, nextStartIndex, totalCount — Pagination state.
  • filterDefaultNames, verbose — Active filter settings.
  • filterByTag — (Present only when filtering by tag.)
  • untagged — (Present only when filtering for untagged functions.)

Compact mode (default) function objects contain:

  • name, address, sizeInBytes
  • tags — Alphabetically sorted list of tags.
  • callerCount, calleeCount — Number of callers and callees. A value of -1 indicates the count timed out.

Verbose mode function objects additionally contain:

  • signature, returnType, callingConvention
  • isExternal, isThunk, isDefaultName
  • parameters — Array of parameter objects with name and dataType.
  • endAddress

Caching

Function listing results are cached for 10 minutes with a maximum of 10 cached entries (keyed by program path and filter settings). The cache is automatically invalidated when the program is closed or when function tags are modified.

get-functions-by-similarity

List functions sorted by name similarity to a search string. Uses Longest Common Substring (LCS) scoring to find functions with names similar to the search term, even when the exact name is unknown. This is useful for locating functions when you know an approximate name or naming convention.

Parameters

Parameter Type Required Default Description
programPath string Yes — Path in the Ghidra Project to the program.
searchString string Yes — The function name or partial name to match against.
filterDefaultNames boolean No true Exclude functions with Ghidra-generated default names.
startIndex integer No 0 Pagination start index (0-based).
maxCount integer No 100 Maximum number of functions to return per page.
verbose boolean No false Return full function details instead of compact summaries.

Return Value

Returns a multi-JSON response. The metadata header includes:

  • searchString — The search term used.
  • startIndex, requestedCount, actualCount, nextStartIndex, totalMatchingFunctions — Pagination state.
  • cacheHit — Whether results were served from cache.
  • resultsTruncated, maxCachedResults — Present when the result set exceeds the 2,000-entry cache limit.

Each function object includes a similarity field (0.0 to 1.0, rounded to 2 decimal places) in addition to the standard compact or verbose fields. Functions are sorted by descending similarity score.

Search Algorithm

The similarity search uses a two-phase optimization strategy:

  1. Phase 1 — Substring pre-filter: Functions whose names contain the search string as a substring are immediately prioritized.
  2. Phase 2 — LCS sorting: Substring matches are sorted first. If fewer than 1,000 substring matches are found, non-matches are also LCS-sorted and appended.

Results are cached for 10 minutes (up to 50 cache entries, 2,000 results per search). The search has a 120-second timeout.

If you know part of a function's name but not the exact spelling or prefix, this tool is more effective than get-functions with manual filtering.

set-function-prototype

Set or update a function's prototype using a C-style signature string. This modifies the function's return type, name, parameters, and optionally its calling convention.

Parameters

Parameter Type Required Default Description
programPath string Yes — Path in the Ghidra Project to the program.
location string Yes — Address or symbol name of the function (e.g., "0x401000" or "main").
signature string Yes — C-style function signature. Example: "int main(int argc, char** argv)". Whitespace around pointer symbols is automatically normalized.
createIfNotExists boolean No true If no function exists at the address, create one before applying the signature.

Return Value

Returns a JSON object containing:

  • success — Boolean indicating the prototype was applied.
  • created — Boolean indicating whether a new function was created.
  • address — Address of the function.
  • parsedSignature — The parsed and normalized signature string.
  • customStorageEnabled, usingCustomStorage — Whether custom storage was needed (e.g., when modifying auto-parameters like this in __thiscall).
  • function — Full function information object.

Error Handling

If a data type in the signature cannot be resolved, the error message includes a helpful hint suggesting you use basic types (e.g., void* instead of FILE*) or ensure the required data type archive is loaded.

Whitespace around pointer symbols is automatically normalized (e.g., "char *func(" becomes "char* func("). You do not need to worry about exact whitespace formatting.

get-undefined-function-candidates

Find addresses that are referenced by code (via calls or data references) but have not been defined as functions. These are likely functions that Ghidra's auto-analysis missed. Use this tool to discover additional code for analysis.

Parameters

Parameter Type Required Default Description
programPath string Yes — Path in the Ghidra Project to the program.
maxCandidates integer No 100 Maximum number of candidates to return.
startIndex integer No 0 Pagination start index (0-based).
minReferenceCount integer No 1 Minimum number of references required for an address to be considered a candidate.

Return Value

Returns a JSON object containing:

  • candidates — Array of candidate objects, each with:
    • address — The candidate address.
    • referenceCount — How many references point to this address.
    • hasCallReference — Whether any reference is a call.
    • hasDataReference — Whether any reference is a data access.
    • sampleReferences — Up to 5 example callers with function names and addresses.
    • memoryBlock — Name of the memory block containing the address.
    • existingSymbol — (Optional) Name of any existing symbol at the address.
  • totalCandidates — Total number of candidates found.
  • referencesScanned — Number of references examined.
  • earlyTermination — (Present if true) Indicates scanning stopped after reaching the 10,000-candidate safety limit.
  • pagination — Object with startIndex, maxCandidates, returnedCount, and hasMore.

Filtering Criteria

The tool scans all CALL and DATA references in the program and excludes addresses that:

  • Already have a defined function.
  • Are in external memory space.
  • Are in non-executable memory.
  • Are in linker-related blocks (.plt, .got, .idata, .edata, extern, external).
  • Do not contain a valid instruction (indicating unanalyzed data).

After identifying candidates, use create-function to define them as functions, then get-decompilation to analyze them.

create-function

Create a new function at the specified address. Ghidra automatically detects the function boundaries based on code flow analysis. The address must contain a valid instruction in executable memory.

Parameters

Parameter Type Required Default Description
programPath string Yes — Path in the Ghidra Project to the program.
address string Yes — Address where the function should start, in hex (e.g., "0x401000").
name string No — Custom name for the function. If omitted, Ghidra assigns a default name (e.g., FUN_00401000).

Return Value

Returns a JSON object containing:

  • success — Boolean indicating the function was created.
  • created — Always true on success.
  • address — Address of the created function.
  • nameWasProvided — Whether a custom name was given.
  • function — Full function information object for the newly created function.

Error Conditions

  • If a function already exists at the address, the operation fails.
  • If no instruction exists at the address, the error suggests running disassembly first.
  • The address must be in executable memory.
  • If a custom name conflicts with an existing symbol, the function is created with Ghidra's default name and a warning is logged.

function-tags

Manage function tags for organizing and categorizing functions in a program. Supports five modes of operation: get, set, add, remove, and list.

Parameters

Parameter Type Required Default Description
programPath string Yes — Path in the Ghidra Project to the program.
mode string Yes — Operation mode. One of: "get", "set", "add", "remove", or "list".
function string Conditional — Function name or address. Required for all modes except "list".
tags array of strings No — Tag names to operate on. Used by "set", "add", and "remove" modes. Empty or whitespace-only tag names are automatically filtered out.

Modes of Operation

Mode Behavior Requires function Requires tags
get Return all tags on the specified function. Yes No
set Replace all tags on the function with the specified tags. Pass an empty array to clear all tags. Yes Yes
add Add the specified tags to the function's existing tags. Yes Yes
remove Remove the specified tags from the function. Yes Yes
list List all tags defined in the program with usage counts. No No

Return Value

For list mode:

  • tags — Array of tag objects with name, count (number of functions using the tag), and optional comment.
  • totalTags — Total number of distinct tags.

For all other modes:

  • mode — The mode that was executed.
  • function — Name of the function.
  • address — Address of the function.
  • tags — Alphabetically sorted array of the function's current tags after the operation.

Modifying tags (set, add, remove) clears both the function info cache and the similarity search cache. This ensures subsequent queries reflect the updated tag state.
