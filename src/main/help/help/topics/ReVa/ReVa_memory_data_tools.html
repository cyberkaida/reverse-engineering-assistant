ReVa - Memory, Data, Strings & Symbol Tools

These tools provide access to program memory layout, raw data reading, defined data inspection, data type application, label creation, string extraction, and symbol enumeration. Together they form the foundation for understanding a binary's data landscape.

Memory Tools

Tools for inspecting the memory layout of a program and reading raw bytes from memory.

get-memory-blocks

Retrieves all memory blocks from the program, including their address ranges, sizes, and permission flags. This provides a high-level view of the program's memory layout.

Parameters

NameTypeRequiredDescription
programPath string Yes Path in the Ghidra Project to the program.

Return Value

FieldTypeDescription
programPathstringPath to the program.
blocksarrayArray of memory block objects.

Each block object contains:

FieldTypeDescription
namestringBlock name (e.g., ".text", ".data", ".bss").
startstringStart address (e.g., "0x00400000").
endstringEnd address.
sizelongBlock size in bytes.
readablebooleanWhether the block is readable.
writablebooleanWhether the block is writable.
executablebooleanWhether the block is executable.
initializedbooleanWhether the block contains initialized data.
volatilebooleanWhether the block is volatile memory.
mappedbooleanWhether the block is memory-mapped.
overlaybooleanWhether the block is an overlay.

Use this tool first to understand the program's memory layout before reading memory with read-memory. The permission flags are particularly useful for identifying code sections (executable), data sections (readable/writable), and uninitialized regions.

read-memory

Reads raw bytes from memory at a specific address. Supports multiple output formats and accepts both hex addresses and symbol names.

Parameters

NameTypeRequiredDefaultDescription
programPath string Yes — Path in the Ghidra Project to the program.
addressOrSymbol string Yes — Address (e.g., "00400000") or symbol name (e.g., "main").
length integer No 16 Number of bytes to read. Must be between 1 and 8192.
format string No "hex" Output format: "hex", "bytes", or "both".

Return Value

FieldTypeDescription
addressstringFormatted start address.
lengthintegerActual number of bytes read.
programPathstringPath to the program.
hexstringHexadecimal representation (present when format is "hex" or "both").
bytesarrayArray of integer byte values 0–255 (present when format is "bytes" or "both").

The maximum read length is 8192 bytes. Attempting to read more will produce an error. For large memory regions, issue multiple reads with advancing addresses.

Data Tools

Tools for inspecting defined data at addresses, applying data types, and creating labels.

get-data

Retrieves the defined data at a specific address or symbol, including its data type, value, and raw bytes. If the address falls within a larger data item, the containing data is returned.

Parameters

NameTypeRequiredDescription
programPath string Yes Path in the Ghidra Project to the program.
addressOrSymbol string Yes Address (e.g., "0x00400000") or symbol name (e.g., "main").

Return Value

FieldTypeDescription
programPathstringPath to the program.
addressstringFormatted address of the data.
dataTypestringName of the data type (e.g., "dword", "string").
lengthintegerLength of the data in bytes.
symbolNamestringSymbol name at this address (if any).
symbolNamespacestringNamespace of the symbol (if any).
hexBytesstringHex representation of the raw bytes.
representationstringGhidra's string representation of the value.
valueTypestringJava class name of the value object.
valuestringString representation of the actual value.

If no data is defined at the given address, an error is returned. If a memory access error occurs when reading bytes, the hexBytes field may contain an error indicator instead of actual byte data.

apply-data-type

Applies a data type to a specific address in the program. The data type is specified using C-style string syntax and resolved through the program's data type archives.

Parameters

NameTypeRequiredDefaultDescription
programPath string Yes — Path in the Ghidra Project to the program.
addressOrSymbol string Yes — Address or symbol name where the data type should be applied.
dataTypeString string Yes — String representation of the data type (e.g., "char**", "int[10]", "dword").
archiveName string No "" (all archives) Name of a specific data type archive to search. If empty, all archives are searched.

Return Value

FieldTypeDescription
successbooleanAlways true on success.
addressstringFormatted address where the type was applied.
dataTypestringName of the applied data type.
dataTypeDisplayNamestringDisplay name of the data type.
lengthintegerLength of the data type in bytes.

The tool automatically clears existing code units at the address before applying the new type. If the data type string cannot be parsed, the error message will suggest using get-data-type-archives to discover available types. This operation runs within a Ghidra transaction and rolls back on failure.

create-label

Creates a named label (symbol) at a specific address in the program. Labels serve as named references that appear in the disassembly listing and can be used as address targets in other tools.

Parameters

NameTypeRequiredDefaultDescription
programPath string Yes — Path to the program in the Ghidra Project.
addressOrSymbol string Yes — Address or existing symbol name where the label should be created.
labelName string Yes — Name for the new label. Must not be empty.
setAsPrimary boolean No true Whether to set this label as the primary symbol at the address.

Return Value

FieldTypeDescription
successbooleanAlways true on success.
labelNamestringName of the created label.
addressstringFormatted address.
isPrimarybooleanWhether the label is the primary symbol.

Labels are created with SourceType.USER_DEFINED, indicating they were explicitly set by a user (or AI assistant) rather than auto-generated by Ghidra's analysis. An address can have multiple labels, but only one can be primary.

Tools for discovering and searching strings within a program's defined data.

Return Value

FieldTypeDescription

get-strings

Retrieves strings from the program with support for three search modes: paginated listing, similarity-sorted search, and regex filtering. Optionally includes referencing function information.

Parameters

NameTypeRequiredDefaultDescription
programPath string Yes — Path in the Ghidra Project to the program.
searchString string No — Sort results by similarity to this string using LCS-based scoring. Mutually exclusive with regexPattern.
regexPattern string No — Filter results using a Java regex pattern. Mutually exclusive with searchString.
startIndex integer No 0 Starting index for pagination (0-based).
maxCount integer No 100 Maximum number of strings to return.
includeReferencingFunctions boolean No false Whether to include the list of functions that reference each string.

Search Modes

ModeActivated ByBehavior
List-all No search parameters Paginated iteration through all strings in memory order.
Similarity searchString provided All strings are sorted by LCS (Longest Common Subsequence) similarity score, then paginated. Best matches appear first.
Regex regexPattern provided Strings are streamed through the regex filter with early termination at maxCount.

Return Value

A multi-result response. The first object contains mode-specific metadata:

FieldTypeDescription
startIndexintegerStarting index used.
requestedCountintegerRequested maximum count.
actualCountintegerNumber of strings returned.
nextStartIndexintegerIndex for fetching the next page.
searchCompletebooleanWhether all matching strings have been returned (similarity/regex modes only).
regexPatternstringThe regex pattern used (regex mode only).

Subsequent objects each describe one string:

FieldTypeDescription
addressstringFormatted address of the string.
contentstringString value (up to 500 characters).
lengthintegerString length in characters.
dataTypestringData type name (e.g., "string", "unicode").
hexBytesstringHex representation of the raw bytes (omitted if truncated).
byteLengthintegerLength in bytes including null terminator.
representationstringGhidra's default representation (omitted if truncated).
truncatedbooleanPresent and true if the string exceeds 500 characters.
fullLengthintegerFull length if truncated.
notestringSuggestion to use get-data or read-memory for full content (if truncated).
referencingFunctionsarrayFunctions referencing this string (if includeReferencingFunctions=true).
referenceCountintegerNumber of referencing functions (if requested).

Limits

  • Maximum string length: 500 characters. Longer strings are truncated with a note suggesting alternative tools for reading the full content.
  • Maximum referencing functions: 100 per string.

The searchString and regexPattern parameters are mutually exclusive. Providing both will produce an error. In similarity mode, referencing functions are only loaded for the paginated subset, not all strings, for performance.

Symbol Tools

Tools for enumerating and browsing symbols defined in the program.

get-symbols

Retrieves symbols from the program with pagination. Each symbol includes its name, address, namespace, type, and source information.

Parameters

NameTypeRequiredDefaultDescription
programPath string Yes — Path in the Ghidra Project to the program.
includeExternal boolean No false Whether to include external symbols.
startIndex integer No 0 Starting index for pagination (0-based).
maxCount integer No 200 Maximum number of symbols to return. Recommended chunks of 200.
filterDefaultNames boolean No true Filter out Ghidra-generated default names.

Return Value

A multi-result response. The first object contains pagination metadata:

FieldTypeDescription
startIndexintegerStarting index used.
requestedCountintegerRequested maximum count.
actualCountintegerNumber of symbols returned.
nextStartIndexintegerIndex for the next page.
totalProcessedintegerTotal symbols examined (before filtering).
includeExternalbooleanWhether external symbols were included.
filterDefaultNamesbooleanWhether default names were filtered.

Subsequent objects each describe one symbol:

FieldTypeDescription
namestringSymbol name.
addressstringFormatted address.
namespacestringSymbol's namespace name.
idlongUnique symbol ID.
symbolTypestringType of symbol: FUNCTION, LABEL, GLOBAL_VAR, etc.
isPrimarybooleanWhether this is the primary symbol at its address.
isExternalbooleanWhether the symbol is external.
isFunctionbooleantrue if the symbol type is FUNCTION.

Default Name Filtering

When filterDefaultNames is true (the default), symbols with the following Ghidra-generated prefixes are excluded:

  • FUN_ — Auto-named functions
  • LAB_ — Auto-named labels
  • SUB_ — Auto-named subroutines
  • DAT_ — Auto-named data
  • EXT_ — Auto-named externals
  • PTR_ — Auto-named pointers
  • ARRAY_ — Auto-named arrays

Use get-symbols-count with the same filter settings to determine the total number of symbols before paginating. The nextStartIndex field in the response indicates the correct starting index for the next page.

Tips & Best Practices

  • Start with get-memory-blocks to understand the program's layout before reading specific addresses.
  • Use get-strings-count and get-symbols-count before their respective listing tools to plan pagination.
  • The similarity search mode in get-strings is particularly useful for finding strings related to a known concept (e.g., searching for "password" will surface strings like "Enter password", "Password incorrect").
  • Use create-label to annotate addresses of interest for easier navigation in subsequent analysis.
  • When applying data types, use get-data-type-archives from the Data Type Tools to discover available types if a type string cannot be resolved.
  • Set filterDefaultNames=false on symbol tools if you need to see all symbols including Ghidra's auto-generated names.
