ReVa - Reverse Engineering Assistant

ReVa (Reverse Engineering Assistant) is a Ghidra extension that connects AI-powered language models to Ghidra through the Model Context Protocol (MCP). It exposes Ghidra's analysis capabilities as a structured set of MCP tools and resources, enabling AI assistants to read, query, and modify program databases on behalf of the analyst.

Rather than replacing the reverse engineer, ReVa acts as a force multiplier. The AI assistant can decompile functions, trace cross-references, search strings, annotate the database, build call graphs, and perform data flow analysis — all through the same Ghidra APIs an analyst would use manually, but at the speed and scale of an automated system guided by natural language.

Design Philosophy

ReVa follows a tool-driven design philosophy. Instead of providing a single monolithic "analyze this binary" command, ReVa exposes a large set of small, focused tools that each perform one well-defined operation. This approach has several advantages:

  • Composability — The AI assistant chains tools together to answer complex questions. A query like "what encryption does this binary use?" might involve searching strings, listing imports, decompiling candidate functions, and tracing data flow — all as separate tool invocations that the assistant orchestrates.
  • Transparency — Each tool call is visible to the analyst. The assistant's reasoning process is observable: which functions it examined, what cross-references it followed, and what conclusions it drew. This makes it possible to verify and correct the assistant's work.
  • Precision — Small tools produce precise, structured output. Rather than generating a wall of unstructured text, each tool returns JSON with well-defined fields that the assistant can reliably parse and act upon.
  • Safety — Tools that modify the program database (renaming variables, applying data types, setting comments) are clearly separated from read-only tools. All modifications occur within Ghidra transactions and can be undone.

ReVa currently provides 78 tools across 19 tool providers, covering the full range of reverse engineering operations from basic navigation to advanced data flow analysis.

Operational Modes

ReVa supports three operational modes, each suited to different workflows. All three modes share the same core tool and resource implementations.

GUI Mode (Ghidra Plugin)

In GUI mode, ReVa runs as a standard Ghidra extension inside the graphical environment. The MCP server starts automatically when Ghidra launches and listens for connections over HTTP.

  • Server runs on http://localhost:8080/mcp/message by default
  • Supports multiple simultaneous programs across CodeBrowser windows
  • Configuration managed through Ghidra's EditTool Options dialog
  • Server persists across tool sessions — closing a CodeBrowser window does not stop the server

This is the recommended mode for interactive analysis, where the analyst works alongside the AI assistant in real time.

Headless Mode

Headless mode runs ReVa without the Ghidra GUI, using PyGhidra to initialize the Ghidra framework programmatically. This mode is suitable for batch processing, CI/CD pipelines, and server deployments.

  • Launched via the reva_headless_server.py script or directly through the Java RevaHeadlessLauncher API
  • Configuration loaded from a .properties file
  • Can create and manage its own Ghidra project
  • Supports random port assignment to avoid conflicts

CLI Mode (Stdio Transport)

CLI mode enables direct integration with command-line AI tools such as Claude Code. The mcp-reva command acts as a stdio-to-HTTP bridge: it reads MCP messages from standard input, forwards them to a local ReVa HTTP server, and writes responses to standard output.

  • Installed via uv tool install reva-cli
  • Registered with Claude Code using claude mcp add ReVa -- mcp-reva
  • Automatically creates a temporary Ghidra project for the session
  • Uses a random port to avoid conflicts with other instances
  • Cleans up temporary projects on exit

Capabilities

ReVa's tools are organized into six categories. Each category contains one or more tool providers, and each provider registers a set of related tools with the MCP server.

Core Analysis

Provider Description Key Tools
Decompiler Decompile functions and work with high-level representations get-decompilation, search-decompilation, rename-variables, change-variable-datatypes
Functions List, search, create, and modify functions get-functions, set-function-prototype, create-function, get-functions-by-similarity
Strings Find and search string references in the program get-strings, get-strings-count
Symbols List and search program symbols get-symbols, get-symbols-count
Cross-References Find references to and from addresses find-cross-references
Memory Inspect memory blocks and read raw memory get-memory-blocks, read-memory

Data and Types

Provider Description Key Tools
Data Read data at addresses and apply data types get-data, apply-data-type, create-label

Structures Parse, create, and apply C structures parse-c-structure, apply-structure, parse-c-header

MCP Resources

In addition to tools, ReVa exposes MCP resources that provide read-only access to program state. Resources use URI-based addressing and support subscriptions for change notifications.

Resource URI Description
Open Programs ghidra://programs Lists all currently open programs with metadata including architecture, compiler specification, and memory size

Security Considerations

By default, the ReVa MCP server binds to 127.0.0.1 (localhost only) and does not require authentication. This means only processes on the local machine can connect to the server.

If the server host is changed to 0.0.0.0 to accept connections from other machines, API key authentication should be enabled to prevent unauthorized access to the program database. See Configuration Options for details.

All program modifications performed through ReVa tools occur within standard Ghidra transactions and can be undone using Ghidra's undo functionality.
